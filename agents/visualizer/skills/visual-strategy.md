# Skill: visual-strategy — 配图策略

## 用途
理解平台特性、分析文案视觉需求、识别素材图片，制定该平台的完整配图方案。

## 输入
- Campaign ID
- 目标平台名称（linkedin / xiaohongshu / gongzhonghao / facebook）
- 该平台已有的 content.md 内容（或 master-copy.md）
- assets/ 目录下的图片清单（如有 image-index.json 优先用它）

## 处理逻辑

### 第一步：理解平台配图特性

| 平台 | 配图数 | 风格 | 尺寸 | 人物面孔 |
|------|-------|------|------|---------|
| LinkedIn | 3-5 张 | 商务、专业、B2B | 1:1 | 非亚洲面孔 |
| Facebook | 1-3 张 | 类似 LinkedIn 但更轻松 | 1:1 | 非亚洲面孔 |
| 小红书 | 6-9 张 | 图片是内容主体 | 3:4 | 亚洲面孔 |
| 微信公众号 | 3-5 张 | 文中插图 | 16:9 | 亚洲面孔 |

**分图建议中每张图必须标注尺寸比例和人物要求。**

### 第二步：理解文案
从 content.md（或 master-copy.md）提取需要可视化的关键要点：
- 哪些段落/观点适合配图？
- 哪些场景需要视觉呈现？

### 第三步：理解素材（带索引机制）

**若 assets/image-index.json 已存在：** 直接用索引中的描述。
**若不存在：** 对 assets/ 中每张图给出语义化描述，建议建立索引。

索引格式：
```json
{
  "product-hero-front.jpg": {
    "original": "image_01.jpg",
    "description": "AI美拍机正面45度产品图，银色机身",
    "tags": ["产品", "正面", "全景"],
    "indexed_at": "2026-04-19"
  }
}
```

### 第四步：制定配图方案
- 该平台需要几张图
- 每张图表达什么要点
- 每张图用哪个素材作为参考
- 哪些图需要 AI 生成（素材缺失时）

## 输出格式（必须严格遵守，多一个字都不行）

整个文件不超过 50 行。**只有以下内容**，没有任何其他章节：

```markdown
# 视觉策略 - [平台名]

## 总体配图策略
[2-3 句话。配几张图、什么风格、图片角色。结束。]

## 分图建议

### 图1：[要点]
- 尺寸：（按平台规则）
- 参考素材：[文件名]
- 画面描述：[一句话]
- 人物要求：（按平台规则）

### 图2：[要点]
- 尺寸：（同上）
- 参考素材：...
- 画面描述：...
- 人物要求：（同上）
```

**保存路径：** `Campaigns/CXXXXXX/[平台名]/visual-strategy.md`

## 禁止添加的内容（违反即失败）
- 禁止写"为什么选这张图"
- 禁止写"项目判断"、"匹配逻辑"
- 禁止写"视觉基调/色彩建议/光影建议/构图原则"
- 禁止写"使用铁律"、"备选图建议"、"不建议的素材"
- 禁止写"版式建议"、"执行原则"、"下一步"
- 每张图的描述不超过 3 行

## 文件长度硬限制
visual-strategy.md 总行数不得超过 50 行。

## 关联产品规则
如果用户消息中提到了关联产品及其图片路径：
- **每张配图的参考素材中都必须包含至少一张产品图**
- 在分图建议的"参考素材"字段中列出产品图路径
