# Skill: brief-writer — Campaign Brief 生成

## 用途
将收集到的所有素材、卖点和图片描述，按标准格式生成 brief.md。

## 前置条件
- 已有 Campaign 编号
- 素材已收集（assets/ 目录下有文件）
- 图片已有语义化描述（不能只写文件名）

## Brief 标准格式（严格遵守，不得增减章节）

```markdown
# Campaign Brief - [项目名称]

## 1. 基础信息
- 项目编号：CXXXXXX
- 所属主题：[产品展示 / 技术幕后 / 行业案例 / 品牌动态]
- 本地路径：Campaigns/CXXXXXX/

## 2. 核心卖点 (USPs)
- [卖点1：准确的技术/商业描述]
- [卖点2：差异化优势]
- [卖点3]

## 3. 深度事实描述 (The Context)
[基于当前选定的主题，详细描述项目的背景、意义及核心事实]
```

## 执行步骤
1. 根据四大内容引擎判断所属主题
2. 从收集的素材中提炼核心卖点（事实为王，不脑补）
3. 撰写深度事实描述（逻辑闭环：这是什么？卖给谁？核心价值？证据在哪？）
4. 输出最终 brief.md 内容

## 严格约束
- **只有 3 个章节**：基础信息、核心卖点、深度事实描述。不得添加视觉资产清单、发布计划、平台角度、配图 prompt 等任何额外内容
- **事实为王**：卖点和描述必须基于素材，不得捏造数据或性能指标
- **图片必须有描述**：如需引用图片必须附带语义化描述，严禁只列文件名
