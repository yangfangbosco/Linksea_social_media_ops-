# Skill: platform-adapter — 平台风格剪裁器

## 用途
将母稿转化为不同性格的推文。**严禁跳过母稿直接从 Brief 创作。**

## 前置条件
- 母稿 `master-copy.md` 已生成并经用户确认
- 用户指定了目标平台

## 支持的平台及风格参数

### 小红书 (xiaohongshu)
| 维度 | 要求 |
|------|------|
| 人格 | 资深体验官 |
| 语气 | 亲和、种草感强、多用 Emoji |
| 关注点 | 颜值、直观体验、生活/工作质量提升 |
| 字数 | 正文 500-1000 字 |
| 标题 | 生成 5 个爆款标题备选，必须含 Emoji |
| 标签 | 提取 10-15 个 Hashtags |
| 图片 | 封面图 + 5-8 张内容图 |
| 排版 | 短段落，Emoji 分段，口语化 |

### LinkedIn (linkedin)
| 维度 | 要求 |
|------|------|
| 人格 | 商务专家 / Founders-led |
| 语气 | 沉稳、专业、第一人称（创始人视角） |
| 关注点 | 行业趋势、商业效率、ROI |
| 字数 | 800-1500 词（英文） |
| 结构 | Hook → 痛点/故事 → 解决方案 → CTA |
| 标签 | 3-5 个行业 Hashtags |
| 配图 | 1 张，1:1 |

### 公众号 (gongzhonghao)
| 维度 | 要求 |
|------|------|
| 人格 | 深度观察者 |
| 语气 | 严谨、逻辑感强 |
| 关注点 | 技术壁垒、完整解决方案、行业标杆案例 |
| 字数 | 2000-4000 字 |
| 结构 | 标题 → 摘要 → 导语 → 核心内容（3-5节）→ 总结 |
| 封面图 | 1 张，16:9，专业科技风 |

### Facebook (facebook)
| 维度 | 要求 |
|------|------|
| 人格 | 社区伙伴 |
| 语气 | 轻松、对话式、鼓励互动 |
| 关注点 | 社区感、互动性 |
| 字数 | 200-500 词（英文） |
| 结构 | 吸睛开头 → 核心内容 → 互动提问 |
| 标签 | 2-3 个 Hashtags |
| 配图 | 1 张，1:1，生活化场景 |

## 处理逻辑
1. 读取 `master-copy.md`
2. 根据目标平台参数进行风格转换：
   - 调整语气和人称
   - 压缩或扩展篇幅
   - 适配排版风格
   - 生成标题（小红书生成 5 个备选）
   - 提取/生成 Hashtags
   - 标注配图位置和 prompt
3. 输出适配后的内容

## 输出
完整平台稿内容，保存路径：`Campaigns/CXXXXXX/[平台名]/content.md`

## 严禁
- **严禁跳过母稿**：必须基于 master-copy.md 进行二次创作
- **严禁语气混淆**：LinkedIn 上不种草，小红书上不学术
- **严禁信息失真**：核心事实必须与母稿一致
