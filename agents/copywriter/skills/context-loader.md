# Skill: context-loader — 项目上下文加载器

## 用途
建立"图片-语义"映射表与事实底座，确保后续生成"言之有物"。

## 输入
Campaign_ID（例如：`C9mQ4x`）和已读到的 brief.md 内容、assets/ 目录文件列表。

## 处理逻辑
1. **解析 Brief**：完整理解 brief.md 内容
2. **扫描素材**：列出 assets/ 下所有文件
3. **建立图片-语义映射表**：将每个文件名与 Brief 中的视觉描述关联

例如：
```
feature_01.png → "展馆场景中数字人一体机的实拍交互图"
hero.jpg → "产品正面45度全景图，白色机身，10.1寸屏幕"
```

## 输出格式

```
📂 项目上下文已加载 — CXXXXXX

📌 所属主题：[产品展示 / 技术幕后 / 行业案例 / 品牌动态]

🎯 核心卖点：
1. [卖点1]
2. [卖点2]
3. [卖点3]

🖼 视觉资产映射：
- feature_01.png → [语义描述]
- hero.jpg → [语义描述]
- ...

📝 事实底座：
[Brief 中的深度事实描述摘要]
```

## 注意事项
- 如果 brief.md 不存在，告知："该项目还没有 Brief，请先 @想得多 完成策划。"
- 如果 assets/ 为空，告知素材还未收集
- 图片-语义映射必须精确，不得脑补不存在的描述
